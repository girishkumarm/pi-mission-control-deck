P0 · DATA LOSS

SQS AUTO_ACKNOWLEDGE — All 17 Consumers

Every SQS consumer in eq-fms-consumers uses Session.AUTO_ACKNOWLEDGE, deleting messages before processing.

// Pattern found in ALL 17 consumers: // eq-fms-consumers/src/.../sqs/ Session session = connection.createSession( false, Session.AUTO_ACKNOWLEDGE ); // Message is deleted from SQS the instant // it is received — BEFORE onMessage() runs @Override public void onMessage(Message message) { try { processPayment(message); } catch (Exception e) { // Message already gone from SQS // No retry, no DLQ, no recovery log.error("[eventMessage] [Exception]", e); } }

P0 · FINANCIAL PRECISION

BigDecimal(double) — 150 Sites, 79 Files

IEEE 754 floating-point contaminates all financial math. new BigDecimal(0.1) = 0.1000000000000000055511151231257827021181583404541015625

// UtilityService.java — found 150 times in 79 files BigDecimal value = new BigDecimal(doubleValue); // Preserves ALL 53 bits of imprecision // ChargesInfoServiceImpl.java — rounding cascade double exchangesTurnOverCharge; // instance var in singleton! double sebiFeesCharge; // shared across threads! Double realisedProfit; // Rs 10,000.50 stored as 10000.5f = Rs 10,000.50 // Rs 99,999.99 stored as 99999.992f = Rs 100,000.00

Impact: Charge shown to user (CEILING) is systematically higher than what's recorded (HALF_UP). Creates phantom revenue gap that never reconciles.

Thread-Unsafe Singleton

ChargesInfoServiceImpl is a @Service singleton with instance variables for exchangesTurnOverCharge and sebiFeesCharge. Two concurrent requests overwrite each other's values → wrong GST calculated for both.

Thread-Unsafe SimpleDateFormat

Shared SimpleDateFormat instances used across threads produce garbled dates. Financial date calculations use ZoneId.systemDefault() — K8s UTC containers misassign IST business days.

Fix: 1) Replace all new BigDecimal(double) with BigDecimal.valueOf(double) (uses String internally). 2) Convert Double fields to BigDecimal in entities. 3) Make charge calculator use local variables. 4) Standardize on HALF_UP. 5) Replace Float ltp with BigDecimal.

NEW: Unbounded Thread Pools

Multiple services use Executors.newCachedThreadPool() or thread counts derived from user input, with no upper bounds. Under load, these can create thousands of threads and crash the JVM.

Timezone Bug

// Financial date calculations throughout codebase LocalDate.now(ZoneId.systemDefault()); // K8s containers run in UTC // IST 00:30 payment → assigned to WRONG business day // Affects settlement dates, trading windows // Fix: Use ZoneId.of("Asia/Kolkata") explicitly

NEW: Thread-Unsafe Singletons

Beyond ChargesInfoServiceImpl, multiple @Service singletons store request-scoped state in instance variables. Under concurrency, one user's data leaks into another's calculation.

Order Recon: Auto-Recovery OFF

// OrderStatusReconScheduler.java List<Order> mismatches = findMismatches(); if (!mismatches.isEmpty()) { // recoverOrders(mismatches); // COMMENTED OUT log.warn("Found {} mismatches", mismatches.size()); } // Orders filled at exchange never appear in app
Fix: 1) Add abstraction layer over RupeeSeed with fallback. 2) Cap thread pools with newFixedThreadPool(Math.min(n, MAX)). 3) Paginate all findAll queries. 4) Use ZoneId.of("Asia/Kolkata"). 5) Enable order reconciliation recovery.
