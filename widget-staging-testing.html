  • widget_handler node skips LLM round-trip (optimization)
  • Calls Widget Service /api/v1/domain/{d}/widgets/transform
  • strip_widget_messages() cleans session history

Issue-1 Impact

What breaks when the Widget Service audience is missing from Keycloak JWT

Cannot View Versions

Widget version dropdown is empty. Users cannot select which version to use.

2

Cannot Load Schema

Widget data schema fails to load. Panel shows "No schema found for this widget version."

3

Cannot Map Params

Input parameter mapping section is completely empty. Cannot configure widget-to-canvas bindings.

Fix: Register the Widget Service as a Keycloak client, add com.paytm.pi.widget-service (or equivalent) to the JWT audience list in the token exchange configuration.
